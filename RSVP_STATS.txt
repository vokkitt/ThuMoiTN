📊 HƯỚNG DẪN THỐNG KÊ RSVP - Khắc Kiệt Graduation Ceremony
===========================================================

📝 TÓM TẮT
---------
- File này hướng dẫn cách xem và quản lý xác nhận tham gia (RSVP) từ khách mời
- Dữ liệu được lưu trữ trong browser's localStorage
- Có thể xuất dữ liệu thành CSV hoặc JSON file để chia sẻ

🚀 CÁCH TRUY CẬP ADMIN DASHBOARD
--------------------------------

1. Mở file admin.html trong browser:
   - Local: file:///Users/vokhackiet/Downloads/ThuMoiTN/admin.html
   - Online: https://your-domain.com/admin.html (sau khi deploy)

2. Sẽ hiện:
   ✅ Tổng số xác nhận
   ✅ Số người sẽ tham gia
   ✅ Số người không tham gia
   ✅ Bảng chi tiết tất cả responses

📥 DOWNLOAD DỮ LIỆU
------------------

Có 3 cách download dữ liệu từ admin dashboard:

1. **Download JSON** (Khuyến nghị để share trên git)
   - Nút: "Tải JSON"
   - Output: rsvp_responses_YYYY-MM-DD.json
   - Format: JSON array, dễ parse
   - Dùng để: Share lên GitHub, backup, analyze

2. **Download CSV** (Dùng Excel/Google Sheets)
   - Nút: "Xuất CSV"
   - Output: rsvp-responses-TIMESTAMP.csv
   - Format: Comma-separated values
   - Dùng để: Mở trong Excel, Google Sheets, hoặc analysis

3. **Copy từ Browser Console**
   - Mở DevTools (F12)
   - Vào tab Console
   - Chạy: getSavedRSVPs()
   - Copy bảng hiện ra

📱 XEM RESPONSES LỤC LỰC
------------------------

Admin dashboard tự động refresh dữ liệu mỗi 5 giây
- Không cần refresh page
- Sẽ tự cập nhật khi có người submit form

🔧 QUẢN LÝ DỮ LIỆU
------------------

Admin dashboard cho phép:

1. ✏️ **Xem chi tiết từng response**
   - Họ tên, email, xác nhận, ghi chú, thời gian

2. 🗑️ **Xóa individual response**
   - Nút "🗑️" ở mỗi hàng
   - Confirm trước khi xóa

3. 🔴 **Xóa tất cả dữ liệu**
   - Nút "Xóa Tất Cả"
   - ⚠️ Hỏi confirm 2 lần (an toàn)
   - Không thể hoàn tác!

📊 GHI CHÚ VỀ STORAGE
---------------------

- Dữ liệu lưu trữ trong **localStorage** của browser
- Khác nhau cho mỗi browser/device
- Không bị xóa khi close browser
- Sẽ bị xóa nếu clear browser data

⚠️ BACKUP DỮ LIỆU
------------------

QUAN TRỌNG: Hãy regularly download JSON/CSV để backup!
Vì:
- localStorage có thể bị xóa khi xóa browser data
- Khác browser/device có dữ liệu khác nhau
- Nếu deploy online, mỗi user có localStorage riêng

💡 BEST PRACTICE
-----------------

1. Hàng ngày (hoặc hàng giờ nếu có nhiều responses):
   - Vào admin.html
   - Click "Tải JSON"
   - Lưu file vào thư mục

2. Upload lên GitHub:
   - Commit file rsvp_responses_YYYY-MM-DD.json
   - Để mọi người xem kết quả tham gia

3. Share trên thiệp/thông báo:
   - "Xem thống kê tham gia: [link đến GitHub]"
   - Hoặc attach file CSV/JSON

🌐 KHI DEPLOY ONLINE
---------------------

Nếu deploy lên hosting (GitHub Pages, Vercel, etc):

1. **Localhost**: localStorage chỉ có dữ liệu local
   - Khách A submit → localStorage của khách A
   - Khách B submit → localhost khác → khác localStorage
   - Không sync được!

2. **GIẢI PHÁP**: Cần backend để sync
   - Option 1: Deploy với backend (Node.js, PHP, Python)
   - Option 2: Dùng Firebase/Supabase
   - Option 3: Dùng service như Formspree/Web3Forms

   Hiện tại admin.html chỉ xem được dữ liệu local browser!

📞 LIÊN HỆ/HỖ TRỢ
-----------------

Nếu có vấn đề:
1. Check browser console (F12) xem có error không
2. Thử clear cache và refresh
3. Try khác browser để test
4. Check rsvp_responses.json format
